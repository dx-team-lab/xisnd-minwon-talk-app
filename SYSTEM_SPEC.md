## 1. 시스템 개요

**서비스명**: 민원 대응 통합 플랫폼 (MinwonTalk)

**목적**: 건설 현장에서 발생하는 민원을 체계적으로 관리·대응하기 위한 사내 지식 공유 플랫폼.  
현장별 민원 진행 현황 추적, 유사 사례 데이터베이스 조회, 대응 방안 가이드, 참고 서식/자료 다운로드 기능을 제공한다.

**주요 사용자**: 건설 현장 관리자(BM), 공사·공무 담당자, 구매팀 등 사내 임직원

**현재 기술 스택** (참고용): Firebase(인증·DB·스토리지) + Next.js  
**신규 시스템 예정 스택**: AWS + Spring Boot 백엔드 + 프론트엔드(별도 결정)

## 2. 전체 화면 구성 및 이동 흐름

### 2.1 화면 목록

| URL 경로 | 화면명 | 접근 가능 역할 |
|----------|--------|--------------|
| `/` | 로그인 / 회원가입 | 누구나 (비로그인) |
| `/forgot-password` | 비밀번호 찾기 | 누구나 |
| `/dashboard` | 홈 메인 | 승인된 사용자 전체 |
| `/dashboard/status` | 민원 진행 현황 | 승인된 사용자 전체 |
| `/dashboard/process` | 민원 대응 절차 | 승인된 사용자 전체 (메뉴 노출은 설정으로 제어) |
| `/dashboard/guides` | 유사사례 및 대응방안 | 승인된 사용자 전체 |
| `/dashboard/references` | 참고자료 | 승인된 사용자 전체 |
| `/dashboard/users` | 사용자 관리 | **관리자(admin)만** |
| `/dashboard/settings/system` | 시스템 설정 | **관리자(admin)만** |
| `/admin/logs` | 시스템 로그 | **관리자(admin)만** |

### 2.2 화면 이동 흐름

```
[비로그인 상태]
  → 로그인/회원가입 페이지 (/)

[회원가입]
  → 이메일 + 비밀번호 + 이름 입력
  → 가입 완료 후 승인 대기 상태(approved=false)로 생성
  → 로그인 시 "승인 대기" 화면 표시 (대시보드 접근 불가)
  → 관리자가 승인 처리 후 정상 접근 가능

[로그인 성공 + 승인됨]
  → 홈 메인 (/dashboard)
  → 상단 헤더 메뉴로 각 화면 이동:
      ├── 민원 진행 현황
      ├── 민원 대응 절차 (설정에서 활성화 시 메뉴 표시)
      ├── 유사사례 및 대응방안
      ├── 참고자료
      └── 설정 [관리자만, 드롭다운]
            ├── 사용자 관리
            ├── 시스템 설정
            └── 시스템 로그
```

## 3. 화면별 상세

### 3.1 로그인 / 회원가입 (`/`)

**구성 요소**
- 좌측: 서비스 소개 문구 + 대표 이미지
- 우측: 로그인/회원가입 전환 폼

**로그인 폼 항목**: 이메일, 비밀번호, 이메일 기억하기(체크박스)  
**회원가입 폼 항목**: 이름, 이메일, 비밀번호  
**추가 링크**: 비밀번호 찾기 페이지 이동, 로그인 ↔ 회원가입 전환

**에러 처리**
- 이메일 형식 오류, 비밀번호 6자 미만, 이미 사용 중인 이메일, 잘못된 인증 정보, 요청 횟수 초과 등 상황별 안내 메시지 표시

---

### 3.2 홈 메인 (`/dashboard`)

**구성 요소**  
두 가지 테마(TypeA / TypeB) 중 시스템 설정에서 선택한 테마로 표시됨.

- **TypeA**: 메인 이미지 표시형 레이아웃 (관리자가 업로드한 이미지 사용)
- **TypeB**: 별도 레이아웃 (확인 필요 — 코드 파일은 존재하나 상세 UI 내용 미확인)

---

### 3.3 민원 진행 현황 (`/dashboard/status`)

**목적**: 전체 현장의 민원 현황을 한눈에 파악

**테이블 컬럼**

| 컬럼 | 내용 |
|------|------|
| 입지 | 지역 유형 배지 (주거/상업/공업/민감) |
| 현장명 | 현장 이름 |
| 공사 진행 단계 | 단계 진행 프로그레스바 + 현재 단계 배지 (착공전/토공/골조/마감/준공) |
| 민원 처리 현황 | 완료 N건 / 진행중 N건 (클릭 시 민원인 목록 팝업) |
| 주요 내용 | 현장 민원 주요 내용 요약 (클릭 시 민원인 목록 팝업) |
| 민원 지도 | 지도 아이콘 버튼 (클릭 시 현장 이미지 팝업) |

**팝업 1 — 민원인 목록**  
해당 현장의 민원인 목록을 표시. 컬럼: 순번+민원인 / 용도 / 소유주 / 진행 상태(완료/진행중)

**팝업 2 — 민원 상세 (민원인 행 클릭 시)**  
- 상단: 현장명, 민원인 정보, 처리 상태 (완료/진행중)
- 단계 진행 스테퍼: 민원 발생 → 민원 대응 → 보상 협상 → 합의 및 집행 (완료 시 전체 체크)
- 단계별 내용 테이블: 각 단계에 입력된 경위/내용 텍스트
- 대응 방안: 적용된 조치방안 목록 (문서 링크 있을 경우 [문서 보기] 링크 함께 표시)
- 유사 사례: 참조 사례 목록 (URL 있을 경우 [문서 보기] 링크 함께 표시)

**팝업 3 — 현장 이미지**  
등록된 현장 이미지를 순서대로 표시. 이미지 클릭 시 새 탭에서 원본 크기로 열림.  
이미지 클릭 시 민원인 목록 팝업으로 전환되는 동작도 지원.

---

### 3.4 민원 대응 절차 (`/dashboard/process`)

**목적**: 표준화된 민원 대응 프로세스 이미지를 사내 공유

**구성 요소**
- 관리자가 시스템 설정에서 업로드한 절차 이미지 1장 표시
- 이미지가 없으면 "등록된 정보 없음" 안내 메시지 표시

> 이 메뉴는 상단 헤더에서 시스템 설정의 `민원 대응 절차 메뉴 활성화` 설정에 따라 표시/숨김이 제어된다.

---

### 3.5 유사사례 및 대응방안 (`/dashboard/guides`)

**목적**: 과거 사례와 유형별 대응 방안을 검색·조회

**상단 필터바**
- 지역/지구 필터: 공업 / 주거 / 민감 / 상업 (다중 선택)
- 단계 필터: 착공전 / 토공 / 골조 / 마감 / 준공 (다중 선택)
- 유형 필터: 교통 / 낙진 / 냄새 / 분진 / 빛 / 소음 / 일반 / 진동 / 파손 (다중 선택)
- 키워드 검색창 (expandable)
- 엑셀 다운로드 버튼 (현재 필터 결과를 .xlsx로 저장 — 유사 사례 / 대응 방안 시트 분리)
- 필터 초기화 버튼

**본문 레이아웃** (좌우 분할)

| 영역 | 비율 | 내용 |
|------|------|------|
| 유사 사례 테이블 | 3/5 | 민원 내용 / 발생시점 / 유형 / 신청인 / 보상금액, 페이지네이션 |
| 대응 방안 테이블 | 2/5 | 구분 / 주요 내용 / 문서 링크(있을 경우) |

**유사 사례 테이블 동작**
- 한 페이지 표시 건수 선택: 10 / 20 / 50 / 100 / 전체
- `상세내용(details)` 필드가 URL인 경우 해당 행 클릭 시 새 탭으로 이동

**대응 방안 테이블 동작**
- 문서 링크(SharePoint URL)가 있으면 [문서 보기] 링크 표시
- 필터(지역/단계/유형/키워드)에 반응하여 실시간 필터링

---

### 3.6 참고자료 (`/dashboard/references`)

**목적**: 민원 대응에 필요한 서식, 가이드라인, 관련 사이트 제공

**메인 테이블 — 문서 양식**

| 컬럼 | 내용 |
|------|------|
| 구분 | 문서명 (예: 민원 일지, 민원 합의서) |
| 언제 사용하나요? | 사용 시기 설명 |
| 누가 작성하나요? | 담당 역할 (BM, CM, 공무 등) |
| 해당 문서는 왜 작성하나요? | 작성 목적 |
| 다운로드 | 양식 다운로드 버튼 + 작성 예시 버튼 |

**다운로드 버튼 동작**
- 파일 1개: 바로 새 탭에서 열기
- 파일 복수 개: 드롭다운으로 개별 파일 선택 또는 ZIP 전체 다운로드

**기본 제공 문서 카테고리** (관리자 미등록 시 기본값으로 표시)
- 민원 일지 / 민원 합의서 / 발파 계획서 및 계측일지 / 소음 측정 일지 / 환경 인허가 / 환경 서약서 및 교육일지 / 살수차·세륜기 운영 일지

**우측 사이드바 — 참고 사이트**
- 관련 기관 및 유용한 외부 링크 목록 (관리자가 등록)
- 각 항목 클릭 시 새 탭으로 이동

**문의하기**
- "구매팀으로 문의하기" 버튼 클릭 → 문의 모달
- 자유 텍스트 입력 후 "보내기" → 관리자에게 문의 접수

---

### 3.7 사용자 관리 (`/dashboard/users`, 관리자만)

**목적**: 사용자 승인, 권한 부여, 삭제 관리

**테이블 컬럼**

| 컬럼 | 내용 |
|------|------|
| 구분 | 본인 여부 표시, 보호된 계정 배지 |
| 이름 | 이름 표시, 관리자만 인라인 편집 가능 |
| 이메일 | 이메일 주소 |
| 최종 접속 | 마지막 접속 일시 (기록 없으면 "기록 없음") |
| 승인 상태 | 토글 스위치 (승인됨 ↔ 대기중) |
| 관리자 권한 | 토글 스위치 (관리자 부여/해제) |
| 관리 | 이름 수정 버튼, 삭제 버튼 |

**제약 사항**
- 보호된 계정(2개): 삭제 버튼 비활성화
- 본인 계정: 삭제 버튼 비활성화

### 3.8 시스템 설정 (`/dashboard/settings/system`, 관리자만)

**목적**: 서비스 운영에 필요한 마스터 데이터 일괄 관리

**탭 구성**

| 대응 방안/유사 사례 | 사례 | 유사 사례 CRUD, 엑셀 가져오기/다운로드 |

| 참고자료/문의 | 참고자료 관리 | 문서 카테고리, 파일 업로드, 참고 사이트 링크 관리 |
| 참고자료/문의 | 문의/요청 관리 | 사용자 문의 목록 조회, 확인 완료 처리 |

### 3.9 시스템 로그 (`/admin/logs`, 관리자만)

**표시 내용**

**로그가 기록되는 동작**
- 현장 등록·수정·삭제
- 민원인 등록·수정·삭제
- 대응 방안(구버전/신규) 등록·삭제
- 유사 사례 등록·삭제
- 조치방안 링크 등록·삭제
- 엑셀 대량 가져오기/내보내기
- 문의 "확인 완료" 처리

## 4. 기능별 상세

### 4.1 인증 — 로그인 / 회원가입 / 비밀번호 찾기

**로그인**
- 이메일 + 비밀번호 입력
- 성공 시 홈 메인으로 이동
- 이메일 기억하기 옵션 제공

**회원가입**
- 이름 + 이메일 + 비밀번호(6자 이상) 입력
- 가입 즉시 `승인 대기(approved=false)` 상태로 생성
- 관리자가 승인해야 대시보드 사용 가능

**비밀번호 찾기**
- 별도 페이지(`/forgot-password`)에서 처리

---

### 4.2 사용자 관리

**승인 처리**
- 관리자가 사용자 목록에서 승인 토글 전환
- 승인됨(ON) → 사용자가 즉시 대시보드 접근 가능
- 대기중(OFF) → 대시보드 진입 차단, 승인 대기 화면 표시

**관리자 권한 부여/해제**
- 관리자가 관리자 권한 토글 전환
- 권한 부여 시 해당 사용자는 설정 메뉴 전체(사용자 관리, 시스템 설정, 시스템 로그) 접근 가능

**이름 수정**
- 관리자만 가능
- 이름 셀에서 인라인 편집, Enter로 저장 / Escape로 취소

**사용자 삭제**
- 삭제 시 사용자 정보 및 관리자 권한 정보 함께 삭제
- 보호된 계정 2개(아래 6장 참조) 및 본인 계정은 삭제 불가

### 4.3 현장 관리 (시스템 설정 > 현장 관리 탭)

**현장 등록/수정/삭제**

등록 항목:
- 지역 유형 (주거/상업/공업/민감)
- 현장명
- 공사 단계 (복수 선택 가능: 착공전/토공/골조/마감/준공)
- 주요 내용 (텍스트)
- 표시 순서 (숫자)
- 이미지 첨부 (최대 5장, JPG/PNG/GIF/WEBP)

삭제 시: 소속 이미지 및 민원인 데이터도 함께 삭제 (캐스케이드 삭제)

**민원인 등록 (현장 하위)**

현장 등록/수정 폼 내에서 민원인을 개별 추가. 등록 항목:
- 순번
- 민원인 (주소 또는 성명)
- 용도 (건물/토지 용도)
- 소유주
- 처리 상태 (완료/진행중)

**민원 상세 입력 (민원인 "상세 입력 열기" 클릭)**
- 현재 진행 단계 선택 (민원 발생/민원 대응/보상 협상/합의 및 집행/완료)
- 각 단계별 내용 텍스트 입력 (민원 발생 내용, 민원 대응 내용, 보상 협상 내용, 합의 및 집행 내용)
- 대응 방안 선택: 조치방안 링크 목록에서 복수 선택
- 유사 사례 직접 입력: 사례 내용 텍스트 + 참조 URL (행 추가/삭제 가능)

엑셀 다운로드: 현재 전체 데이터를 .xlsx로 내보내기  
전체 삭제: 확인 모달 후 전체 데이터 삭제

등록 항목:

- 지역/지구 (전체/공업/주거/민감/상업)
- 단계 (전체/착공전(철거)/토공/골조/마감/준공)
- 유형 (복수 선택: 교통/낙진/냄새/분진/빛/소음/일반/진동/파손)

엑셀 가져오기 컬럼: `구 분 / 주요 내용 / 문서 링크 / 지역/지구 / 단계 / 유형`

### 4.5 유사 사례 관리 (시스템 설정 > 사례 탭)

등록 항목:
- 현장명 / 지역 / 발생시점(단계) / 유형(복수)
- 민원 내용 / 신청인 / 발생 일시
- 진행경과 (접수/진행/종결)
- 보상방식 (미보상/과태료/시설보수/안전진단/예방활동/현물보상)
- 요구사항 (복수 선택: 정신적피해보상/영업피해보상/재산피해보상/분쟁조정/대인피해보상/행정처분)
- 보상금액 (보상방식이 "미보상"이면 입력 비활성화)
- 상세내용 및 관련 링크 (URL 입력 시 대시보드에서 링크로 자동 변환)

엑셀 가져오기 컬럼: `지역 / 단계 / 유형 / 민원인 / 요구사항 / 진행경과 / 보상방식 / 보상금액 / 상세내용 / 발생일시`  
엑셀 다운로드 및 전체 삭제 지원

### 4.6 조치방안 링크 관리 (시스템 설정 > 조치방안(링크) 탭)

**목적**: 민원인 상세 입력에서 "대응 방안"으로 선택 가능한 항목 목록을 관리

등록 항목:
- 조치방안명 (예: 통제원 배치)
- URL (SharePoint 문서 링크)
- 관련 유형 (복수 선택)

엑셀 가져오기 컬럼: `제목 / 내용(URL) / 유형1 / 유형2 / 유형3 / 유형4`  
엑셀 다운로드 및 전체 삭제 지원

### 4.7 참고자료 관리 (시스템 설정 > 참고자료 관리 탭)

**문서 카테고리 관리**
- 카테고리(문서명) 추가·수정·삭제
- 각 카테고리별 항목: 문서명 / 사용 시기 / 담당자 / 목적
- 각 카테고리에 양식 파일 및 작성 예시 파일 업로드 (복수 가능)

**참고 사이트 관리**
- 사이트명 + URL 추가·삭제
- 참고자료 화면 우측 사이드바에 표시됨

### 4.8 문의 관리 (시스템 설정 > 문의/요청 관리 탭)

**목적**: 사용자가 참고자료 페이지에서 제출한 문의를 관리자가 확인·처리

**관리자 동작**
- 문의 목록 조회 (최신순): 문의 내용 / 작성자(이름+이메일) / 제출 일시 / 처리 상태 표시
- "확인 완료" 버튼 클릭 → 상태가 "대기중 → 완료"로 변경

**알림 배지**: 헤더 우측 종 모양 아이콘은 **관리자에게만 표시**되며, 대기 중 문의 건수를 실시간으로 표시함. 클릭 시 시스템 설정 > 문의/요청 관리 탭으로 이동.

## 7. 비즈니스 규칙 모음

1. **회원가입 후 관리자 승인 필수**: 가입 즉시 `approved=false`로 생성. 관리자가 승인 전까지 대시보드 진입 불가. 미승인 사용자에게는 승인 대기 화면만 표시되며 로그아웃 및 새로고침 버튼만 제공됨.

2. **민원 대응 절차 메뉴 온/오프**: 시스템 설정의 `민원 대응 절차 메뉴 활성화` 값에 따라 상단 헤더 메뉴에 표시/숨김 제어. 비활성화 시 URL 직접 접근은 별도 차단 로직 없음 (확인 필요).

3. **현장 이미지 최대 5장**: 현장 1개당 이미지 최대 5장 제한. 초과 시 추가 업로드 비활성화.

4. **유사 사례 상세내용 URL 자동 링크**: 유사 사례의 `details` 필드에 `http`로 시작하는 값이 입력된 경우, 유사사례 테이블에서 해당 행 클릭 시 새 탭으로 이동.

5. **보상금액 입력 제한**: 유사 사례 등록 시 보상방식이 "미보상"으로 선택된 경우 보상금액 입력 필드 비활성화.

6. **보호된 계정 삭제 불가**: 지정된 2개 계정(6.4 참조)은 사용자 관리 화면에서 삭제 버튼이 비활성화됨.

7. **본인 계정 삭제 불가**: 현재 로그인한 계정은 본인이 삭제할 수 없음.

8. **활동 로그 조회 건수 제한**: 시스템 로그 화면에서 최근 100건만 표시.

9. **문의 처리 단방향**: 현재 시스템에서 문의 상태는 "대기중 → 확인 완료" 단방향으로만 변경 가능. 완료 처리 취소 기능 없음.

10. **대응 방안 구버전/신규 버전 병존**: 구버전(`responseGuides`)과 신규버전(`responsePlansV2`)이 모두 유지되고 있으며, 유사사례 및 대응방안 화면에는 신규버전 데이터만 우측에 표시됨. 구버전 데이터는 시스템 설정 > 대응 방안 탭에서만 관리.

11. **민원인 순번 중복 불가**: 동일 현장 내 민원인 순번(`number`)은 중복 불가. 저장 시 중복 검증 후 오류 메시지 표시.

12. **홈 테마 설정**: 시스템 설정 > 메인 이미지 탭에서 TypeA / TypeB 중 선택. 전체 사용자의 홈 화면에 즉시 반영.

## 8. 새 시스템 개발 시 고려사항 및 개선 요청

### 8.1 개선 요청 — 문의 기능 양방향 소통

**현재 상태**: 일반 사용자가 문의를 제출하면 관리자가 "확인 완료" 버튼으로 처리 상태만 변경 가능. 관리자의 답변이 문의자에게 전달되지 않으며, 문의자도 처리 결과를 확인할 방법이 없음.

**새 시스템 개발 요청**: 문의자와 관리자 간 **양방향 소통**이 가능하도록 개발 요청.
- 관리자가 문의에 **답변을 작성**하여 문의자에게 전달하는 기능
- 문의자가 답변을 확인하고 **재문의**를 남길 수 있는 기능
- 문의자가 본인의 문의 내역 및 처리 상태를 확인할 수 있는 **"내 문의 내역"** 화면

---

### 8.2 역할 체계 재설계 검토

현재 시스템은 사용자 프로필의 `role` 필드와 별도 권한 컬렉션(roles_admin)이 이중으로 존재하며, 실질적인 화면 접근 제어는 권한 컬렉션 기준으로 동작함. 새 시스템 설계 시 역할 관리 방식을 단일 구조로 통합 검토 권장.

---

### 8.3 민원 대응 절차 메뉴 URL 직접 접근 차단 여부

현재 시스템에서 "민원 대응 절차" 메뉴는 설정 값으로 헤더 표시/숨김만 제어됨. 설정 비활성화 상태에서 URL 직접 입력 시 접근 가능 여부가 코드상 명확하지 않음. 새 시스템에서는 메뉴 비활성화 시 URL 직접 접근도 차단하도록 처리 권장.

---

### 8.4 대응 방안 구버전/신규버전 통합 검토

현재 구버전(`대응 방안`)과 신규버전(`대응 방안 V2`) 두 종류가 병존하며, 실제 화면에는 신규버전만 노출됨. 새 시스템 개발 시 **신규버전 V2 기준으로 단일화** 권장. 구버전 데이터 마이그레이션 또는 폐기 여부는 운영팀과 협의 필요.

---

### 8.5 활동 로그 조회 범위 확장

현재 시스템은 최근 100건으로 제한됨. 새 시스템에서는 날짜 범위 검색, 수행자 필터, 액션 유형 필터 등 조회 기능 확장을 검토 권장.

---

### 8.6 이미지 저장 방식 변경

현재 현장 이미지는 Base64 인코딩 문자열로 저장됨. 새 시스템에서는 AWS S3 등 오브젝트 스토리지에 파일로 저장하고 URL 참조 방식으로 변경 권장.
